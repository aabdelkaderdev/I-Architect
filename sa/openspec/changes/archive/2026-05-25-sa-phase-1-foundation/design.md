## Context

The Scoring Agent (SA) evaluates architecture models generated by the Requirements Analysis Agent (RAA) and Architecture Generation Agent (AGA). The SA requires a sequential pipeline using LangGraph. This phase (Phase 1) focuses on setting up the foundation: data ingestion (Node 1) and basic deterministic output (Node 5), with mocked nodes in between.

## Goals / Non-Goals

**Goals:**
- Create the `sa/` package structure and standard LangGraph files (`runner.py`, `graphs/core.py`, `state/schemas.py`, `state/models.py`).
- Implement Data Preparation (Node 1) in `nodes/preparation.py`.
- Implement deterministic Report generation (Node 5) in `nodes/report.py` producing `ScoringReport` JSON.
- Provide mocked Nodes 2, 3, and 4 to allow end-to-end execution.

**Non-Goals:**
- LLM integrations for scoring.
- Parsing of PlantUML strings.
- Actual execution of SAAM methodology.

## Decisions

- **LangGraph StateGraph**: We will use a strictly linear sequential StateGraph using the current LangGraph API. The graph will be wired using `builder.add_node()` and `builder.add_edge()` sequentially from `START` to Node 1, through Node 5, and finally to `END`.
- **State Schemas**: State will be typed using `typing.TypedDict` for LangGraph compatibility, matching `SA_PRD.md` Appendix B exactly.
- **Data Ingestion**: Node 1 will traverse the flat list of entities and construct a hierarchical tree and a traceability matrix, storing them in the state for deterministic use by other nodes.
- **Mock Nodes**: Nodes 2, 3, and 4 will be simple functions that inject 0 points and empty reasoning strings into the state. Node 5 will output a stubbed markdown report.
- **Compilation**: The graph will be built and instantiated via `builder.compile()`.

## Risks / Trade-offs

- [Risk] Mock nodes might cause schema divergence if real nodes need different inputs later.
  → Mitigation: The State schema defined in Phase 1 will strictly follow the final schema in `SA_PRD.md`.
