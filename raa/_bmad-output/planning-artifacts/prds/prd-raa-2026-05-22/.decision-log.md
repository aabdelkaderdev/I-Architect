### [2026-05-22] Review Gate Timeout Removal
- **Decision:** Removed the 300-second review gate timeout. The RAA module will wait indefinitely when a LangGraph interrupt is triggered.
- **Context:** If the user wants autonomous execution, they must set the configuration flag (e.g. `review_mode = autonomous`) at pipeline invocation, which bypasses the interrupt entirely.
- **Status:** Review gate behavior locked.

- **Decision:** Locked the quality-architecture matrix file path to a project constant named `matrix.json` located at the root of the RAA module workspace.
- **Context:** Rather than passing the matrix dynamically or resolving it from general inputs, the RAA-B pattern-driven subgraph will read directly from `matrix.json`.
- **Status:** Configuration paths locked.

### [2026-05-22] Local Embedding Model Cache Path
- **Decision:** Configured the FastEmbed embedding model cache directory to use the local path `../models` (fully resolved as `/home/delatom/I-Architect-3cf20c60d77417e9febe099eeb91bc78227ce89f/models`).
- **Context:** This ensures offline execution and prevents RAA from attempting to download embedding models from the internet during runtime.
- **Status:** Cache paths locked.

### [2026-05-22] Input Schema and Orchestration Config
- **Decision:** Defined that all paths (including SQLite database and checkpointer files) and runtime configurations will be passed by the orchestrator via the input schema of the state graph.
- **Context:** RAA's input schema will inherit configuration parameters from the parent pipeline, aligning with the pattern observed in ARLO's input schema.
- **Status:** Interface contract locked.

### [2026-05-22] Local Model Cache Error Handling
- **Decision:** If the required FastEmbed model is missing from the local cache folder (`../models`), RAA must throw an explicit exception declaring that the model files are non-existent.
- **Context:** Prevents silent downloads or unexpected internet access in isolated execution environments.
- **Status:** Error handling behavior locked.

### [2026-05-22] Latency Target Removal
- **Decision:** Removed the definitive duration target per batch (SM-C2) to accommodate local model backends (e.g. Ollama) whose speed is highly hardware-dependent.
- **Context:** The pipeline will run without enforcement of a latency deadline, waiting until execution completes.
- **Status:** SLA policy locked.
